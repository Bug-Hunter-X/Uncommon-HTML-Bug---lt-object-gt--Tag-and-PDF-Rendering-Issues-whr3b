The solution involves ensuring that the type attribute within the <object> tag is correctly set to "application/pdf" to specify that the embedded content is a PDF file. This tells the browser how to handle the content and should resolve rendering issues.

<object data="document.pdf" type="application/pdf" width="800" height="600"><p>Your browser doesn't support embedded PDFs.</p></object>

Furthermore, ensure that the path to your PDF ("document.pdf") is correct. Double-check the MIME type handling by your server as well, to ensure that it is correctly serving PDFs with the application/pdf MIME type. If you're using a webserver like Apache or Nginx, verify your configuration file to make sure that PDF files are correctly mapped to the 'application/pdf' MIME type.
